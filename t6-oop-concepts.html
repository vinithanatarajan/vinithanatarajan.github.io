About Blocs, Procs, and Lambdas

What are blocks, procs, and lambdas? How do they work?

# Block Examples

[1,2,3].each { |x| puts x*2 } # block is in between the curly braces

[1,2,3].each do |x|

puts x*2 # block is everything between the do and end

end

# Proc Examples

p = Proc.new { |x| puts x*2 }

[1,2,3].each(&p) # The '&' tells ruby to turn the proc into a block

proc = Proc.new { puts "Hello World" }

proc.call # The body of the Proc object gets executed when called

# Lambda Examples

lam = lambda { |x| puts x*2 }

[1,2,3].each(&lam)

lam = lambda { puts "Hello World" }

lam.call

Blocks, Procs and Lambda looks like these are all very similar, but there are differences which is discussed below.

let's discuss about Blocks and Procs first

  • Procs are objects, blocks are not
  • A proc (notice the lowercase p) is an instance of the Proc class.

    • p = Proc.new { puts "Welcome to EDA" }

    This lets us call methods on it and assign it to variables. Procs can also return themselves.

    p.call # prints 'Welcome to EDA'

    p.class # returns 'Proc'

    a = p # a now equals p, a Proc instance

    p # returns a proc object #

    A block is just part of the *syntax* of a method call. It doesn't mean anything on a standalone basis and can only appear in argument lists.

    { puts "Welcome to EDA"} # syntax error

    a = { puts "Welcome to EDA"} # syntax error

    [1,2,3].each {|x| puts x*2} # only works as part of the syntax of a method call

  • At most one block can appear in an argument list
  • Multiple procs can be passed to methods.

    • Now let's discuss about Procs and Lambdas

  • They are both Proc objects.

    • proc = Proc.new { puts "Welcome to EDA" }

    lam = lambda { puts "Welcome to EDA" }

    proc.class # returns 'Proc'

    lam.class # returns 'Proc'

    However, lambdas are a different 'flavor' of procs. This slight difference is shown when returning the objects.

    Below are the key differences.

  • Lambdas check the number of arguments, while procs do not

  • lam = lambda { |x| puts x } # creates a lambda that takes 1 argument

    lam.call(2) # prints out 2

    lam.call # ArgumentError: wrong number of arguments (0 for 1)

    lam.call(1,2,3) # ArgumentError: wrong number of arguments (3 for 1)

  • In contrast, procs don't care if they are passed the wrong number of arguments.
  • If the proc requires an argument but no argument is passed then the proc returns nil.
  • If too many arguments are passed than it ignores the extra arguments.

    • proc = Proc.new { |x| puts x } # creates a proc that takes 1 argument

    proc.call(2) # prints out 2

    proc.call # returns nil

    proc.call(1,2,3) # prints out 1 and forgets about the extra arguments

    proc_test # calling proc_test prints nothing
