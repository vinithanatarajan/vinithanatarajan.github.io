Enumerable Method

"Programmers often feel joy when they can concentrate on the creative side of programming, so Ruby is designed to make programmers happy" - said Yukihiro Matsumoto (Matz)

As a beginner it took me a long time to figure out what Enumerable module was and what one might use it for. But later the features provided by the Enumerable module impressed me. Thanks to David A. Black. David teaches Ruby to beginners not only by showing them what enumerable can do, but also making them implement their own version of it which is a very good exercise.

The Enumerable module has a bunch of methods that work with collections. Let's take a look at some of the Enumerable methods.

  • A class 'including' or 'mixing-in' Enumerable must define the 'each' method
  • Yielded items from #each method empower the collection awareness for the class
  • The #collect/map method executes the provided block to all of the values yielded by #each .The #group_by method takes a block and returns a hash, with the returned value from the block set as the key.
  • The #find method finds the item that matches the condition supplied by the block.
  • The #grep method searches for members of the collection according to a pattern... pattern matching. It uses the ===operator for pattern matching.

    • Now lets take a look at one of the enumerable method #map

    map is simply a function that builds up a new array by taking each element and replacing it with the return value of the block you provide to it

    Example:

    names =%w{Sam Dara Jacques}

    => ["Sam", "Dara", "Jacques"]

    names.map {|name| name.downcase}

    => ["sam", "dara", "jacques"]

    In the above example we map one array to another, convert each element by some rule. Think of it as a transformation method, that applies the block as a transformation. It always returns a new array with transformation applied.

    Let's see another example to convert a hash to an array of arrays using map

    fruits = {"Apple" => 20, "Strawberry" => 25, " mandarin" => 30}

    =>{"Apple" => 20, "Strawberry" => 25, " mandarin" => 30}

    fruits.map

    => [["Apple", 20], ["Strawberry", 25], ["Mandarin", 30]]

    In the above method When the block is omitted, map uses this implied block: {|item| item}, which means when applied on an hash without a block, map returns an array containing a set of two-item arrays, one for each key/value pair in the hash. For each two-item array, item 0 is the key and item 1 is the corresponding value.
