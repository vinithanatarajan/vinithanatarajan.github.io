Ruby Hashes vs. JavaScript Objects

Ruby Hashes

Ruby Hashes are collection of unique keys and their values, it allows us to use any object type. Hashes enumerate their values in the order that the corresponding keys were inserted.

A hash can be created in two basic ways: with the new keyword or with the hash literal.

fruits = Hash.new # a hash object is created

fruits[1] = "Apple"

fruits[2] = "Mango"

Two key-value pairs are added to the hash object. The numbers 1, 2 are the keys to the hash. The keys are placed inside the square brackets. The names are the values that belong to the keys.

puts fruits

The puts method prints the string representation of the hash to the console.

{1=>"Apple", 2=>"Mango"}

From the output we can see the literal representation of the names hash. A hash is bounded by curly brackets. The keys and the values are paired with the => characters.

JavaScript Objects

Java script objects are similar to Ruby's hash They are declared as variables which is a way of organizing key/value pairs. JS object is a collection of properties that reference other objects. We can create simple objects and write properties.

For example,

var dog = {color: "grey", name: "Tom", age: "7"};

To access the value of an associated key, you can call that on the variable name:

dog.name returns "Tom".

You can also easily delete a key/value pair by writing delete dog.name.

To declare a new key/value pair, write: dog.gender = "male";.

New data point is added to the end of the Object.

In JavaScript, almost everything is an object. All primitive types except null and undefined are treated as objects. They can be assigned properties, and they have all characteristics of objects.
